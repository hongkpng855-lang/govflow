GovFlow Website — Technical Spec for AI Developer
Project: GovFlow (Hong Kong Government Process Guide Platform)
Your client: A 3-person HK company owner who survived a 3-month shareholder transfer nightmare.
Your role: Build the web application. Business strategy and content are handled by another AI.
Stack preference: Modern, simple, fast to build, easy to maintain by a non-technical founder.

1. Project Overview

Build a single-page application (or multi-page static site) that presents government/bureaucratic processes as interactive, visual step-by-step guides.

MVP scope: ONE process only — Shareholder Transfer for Hong Kong Companies.

The core interaction:

• User sees a visual flowchart/timeline of steps
• User clicks any step → sees explanation + required documents
• User clicks any document → sees a demo/example image + downloadable template
• First 2 steps are fully visible (free), remaining steps show locked/preview state until payment

2. Technology Stack (Recommended)

Option A: Next.js 14 (App Router) + Tailwind CSS + shadcn/ui

• Why: SEO-friendly, static export possible, component ecosystem, good defaults
• Deployment: Vercel (free tier) or Cloudflare Pages

Option B: Astro + Tailwind CSS

• Why: Even simpler, zero JS by default, excellent for content-heavy sites
• Deployment: Cloudflare Pages / Vercel / Netlify

Option C: Pure HTML + Tailwind CDN + Alpine.js

• Why: Zero build step, easiest for non-technical founder to tweak later
• Deployment: Any static host (GitHub Pages, Cloudflare Pages)

Recommendation: Start with Option C for fastest MVP, migrate to Option A if traction proves.

5. UI/UX Requirements

Visual Style

• Clean, trustworthy, government-adjacent but modern
• Color palette: Navy blue + white + subtle gold accent (trust, professionalism)...

• Avoid: playful illustrations, overly bright colors
• Typography: Clear, readable Chinese fonts (Noto Sans TC or system-ui)

Key Interactions

1. Step navigation: Click step number → smooth scroll to content
2. Document cards: Hover lift effect, click to expand overlay
3. Lock/paywall: Blurred content preview + "解鎖" button (not annoying popups)
4. Mobile-first: Most users will be on phone, checking while at government offices

Accessibility

• All images have alt text
• Keyboard-navigable
• Chinese text must be fully supported (no tofu characters)

15. Constraints & Non-Goals

Do NOT build:

• User accounts / authentication (overkill for MVP)
• Custom CMS (use JSON + Git for now)
• Video hosting (use images + text)
• Multi-language (start with 繁體中文 only)
• Complex admin dashboard

Keep it simple. The goal is to validate if people will pay $149 for a guide. Anything that delays launch by more than 1 week is unnecessary.

11. Success Criteria

Website is ready when:

• Homepage loads in < 2 seconds
• Shareholder transfer flow is fully clickable on mobile
• Steps 1-2 show real content (or good placeholders)
• Steps 3-7 show lock state with clear payment CTA
• Payment link works and redirects correctly
• Founder can edit process content without touching code (via JSON)
